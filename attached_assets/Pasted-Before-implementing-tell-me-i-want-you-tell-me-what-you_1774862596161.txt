14. Shared Presentations

### Important product behavior
- User must first choose the source(s)
- User must then fill a presentation brief:
  - title
  - audience
  - objective
  - tone
  - deck type
  - target length
  - design style
  - free-text instruction
- AI must generate an outline/storyboard first
- User must review and refine the outline before full draft generation
- Only after outline approval should the presentation draft be generated
- User must be able to refine:
  - full deck
  - section
  - single slide
- User must be able to save, share, publish, and export presentations
- Support PPTX and PDF export if feasible
- Keep version history
