Build a full-stack web application called **Performo AI**.

Performo AI is a role-based business performance and PMO management platform for SMEs and mid-sized companies.

The application must have:
1. **Admin / Data Owner side** for uploading and maintaining backend data
2. **Executive side** for CEO / GM users to view dashboards, progress, risks, and summaries
3. **Role-based authentication and access control**

The platform combines:
- KPI Management
- PMO / Initiative Tracking
- Action Tracker
- Monthly Review Copilot
- Executive Dashboards

====================================================
1. PRODUCT PURPOSE
====================================================

Create a professional SaaS web app that allows operational users to maintain business data, while leaders such as CEO and GMs can log in to view high-level progress and dashboards.

Core idea:
- Admins upload and manage data
- CEO / GMs view clean dashboards and progress
- AI generates management summaries and insights

The app should feel practical, premium, and ready as a real MVP.

====================================================
2. USER ROLES
====================================================

Implement these roles:

### 1. Super Admin
Can:
- manage all users
- manage all companies
- manage all departments
- configure KPI master data
- upload and edit all KPI actuals
- manage meetings, actions, initiatives, and reviews
- assign permissions
- view all dashboards

### 2. Admin / Data Owner
Can:
- maintain KPI master data for assigned company/departments
- upload KPI actuals
- create and update meetings
- create and update action items
- update initiative / project progress
- enter monthly review notes
- generate AI summaries
- view dashboards for allowed company/departments

### 3. Executive Viewer
This includes CEO and GM users.

Can:
- view dashboards
- view KPI progress
- view overdue actions
- view initiative / PMO progress
- read AI summaries
- filter by department / period / business unit
- optionally add comments or mark items as reviewed

Cannot:
- edit master data
- upload operational data unless explicitly allowed

### 4. Executive Subtypes
Support these user types under executive:
- CEO: can view all departments and all company data
- GM: can view only assigned departments or business units
- Department Head: optional, can view only own department
