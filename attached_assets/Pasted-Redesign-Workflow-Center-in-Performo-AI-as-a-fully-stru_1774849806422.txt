Redesign Workflow Center in Performo AI as a fully structured, highly user-friendly operational workflow module with the following primary structure:

Workflow Center
├─ Recurring Tasks
├─ Service Desk
├─ Licenses
├─ Certificates

Each of these 4 workflow areas must have its own:
- configuration
- data entry / record creation
- operational views
- detailed record pages
- analytics
- tables / filters / exports
- reminders / automations
- comments / activity history

Important:
- Use only Performo AI branding
- Do not mention any third-party product names anywhere in the UI
- Make the UI extremely user-friendly, premium, intuitive, and business-friendly
- This must feel like a polished workflow operating system, not a basic forms module

====================================================
1. CORE PRODUCT LOGIC
====================================================

Workflow Center must be structured around 4 dedicated workflow types:

1. Recurring Tasks
2. Service Desk
3. Licenses
4. Certificates

Each workflow type must follow the same product lifecycle:

1. Config
2. Record creation / data entry
3. Operational views
4. Detailed record view
5. Analytics

This must be implemented consistently for all 4 workflow types.

====================================================
2. GLOBAL NAVIGATION STRUCTURE
====================================================

Workflow Center navigation should be reworked to clearly reflect this structure:

Workflow Center
- Home / Overview
- Recurring Tasks
- Service Desk
- Licenses
- Certificates
- Workflow Builder / Configuration
- Workflow Analytics
- My Requests / Assigned to Me
- Automations / Rules

If needed, each workflow type can have its own internal tabs:
- Overview
- Config
- Records
- Analytics

====================================================
3. VERY IMPORTANT DATA MODEL REQUIREMENT
====================================================

For all workflow types, add support for:
- Assigned To Name
- Assigned To Email Address

This is critical because automation rules must be able to send email reminders and notifications.

Specifically:
- Recurring Tasks must store assigned person name and email
- Service Tickets must store assigned person/team contact name and email
- Certificates must store candidate/holder name and email
- Licenses must store responsible owner/renewal owner name and email

This data must be visible in:
- configuration
- record creation forms
- detailed record view
- automations/rules
- analytics where relevant
