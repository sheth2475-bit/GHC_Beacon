Redesign Analytics Studio in Performo AI to follow a data-first, search-driven analytics workflow.

The goal is to create a self-service analytics experience where users:
1. upload data
2. configure the data model
3. ask questions in natural language
4. receive the best-fit visual automatically
5. save each result as an Insight
6. pin selected Insights to a dashboard
7. publish the dashboard
8. refresh it later with new data uploads

Important:
- Use only Performo AI branding in the UI
- Do not mention any third-party product names anywhere in the application
- This must be a real connected workflow, not a mock concept

====================================================
1. NEW PRODUCT LOGIC
====================================================

Replace the earlier dashboard-first flow with this new logic:

Dataset -> Column Configuration -> Formula & Aggregation Setup -> Ask Questions -> Generate Insight -> Save or Pin Insight -> Compose Dashboard -> Publish -> Refresh with new data

Dashboards must be assembled from pinned Insights, not only generated as a full dashboard in one step.

====================================================
2. CORE USER EXPERIENCE
====================================================

The user journey should be:

1. User uploads Excel data
2. Application reads sheets and columns
3. Application asks user to classify each column:
   - Dimension
   - Measure
   - Date
   - Ignore

4. If a column is a Measure, application asks:
   - default aggregation:
     - sum
     - average
     - count
     - min
     - max
   - format:
     - number
     - currency
     - percentage
   - whether the user wants to create derived formulas using this measure

5. If a column is a Date, application asks:
   - date format
   - whether it should support year / quarter / month / week / day grouping

6. User can create formula fields such as:
   - Gross Margin % = (Revenue - Cost) / Revenue
   - Attrition Rate = Exits / Avg Headcount
   - Revenue per Employee = Revenue / Headcount

7. Once the dataset is configured, user can ask questions in natural language such as:
   - Total Sales
   - Revenue by Region
   - Sales by Month
   - Attrition Rate by Department
   - Top 5 branches by profit
   - Show customer complaints trend this year
   - Compare revenue this month vs last month

8. The application should interpret the question and automatically recommend the best-fit visual.

9. Each generated result should be treated as an Insight.

10. The user can:
   - Save Insight
   - Pin Insight to Dashboard
   - Edit Insight
   - Ask follow-up questions
   - Create another Insight

11. The user gradually assembles a dashboard by pinning selected Insights.

12. Once satisfied, the user can:
   - Save Dashboard as Draft
   - Publish Dashboard
   - Share Dashboard based on permissions

13. Later, the creator can re-upload updated data for the same dataset and refresh the Insights and Dashboard while preserving mappings, formulas, and layout.

====================================================
3. BEST-FIT VISUALIZATION LOGIC
====================================================

Build automatic visualization recommendation logic.

When the user asks a question, the application should analyze:
- number of measures
- number of dimensions
- presence of a date column
- cardinality of categories
- whether user is asking for total, comparison, trend, share, ranking, or distribution

Then recommend the best-fit visual.

Examples:

A. KPI Card
Trigger when:
- user asks for a single total, count, average, or headline metric
Examples:
- Total Sales
- Total Headcount
- Average Response Time
- Attrition Rate

B. Line Chart
Trigger when:
- question includes a time element
Examples:
- Sales by Month
- Complaints trend by week
- Revenue over time

C. Bar / Column Chart
Trigger when:
- comparing values across categories
Examples:
- Revenue by Region
- Profit by Product
- Attrition by Department

D. Pie / Donut Chart
Trigger when:
- showing share of whole and category count is small
Examples:
- Revenue share by region
- Employee mix by category

E. Table
Trigger when:
- user asks for detailed records, ranked lists, or tabular breakdown
Examples:
- Top 10 customers by sales
- Show branch-wise revenue and profit

F. Combo / Mixed Visual
Trigger when:
- user compares trend plus target / benchmark if supported

The user should still be able to override the recommended chart type manually.
