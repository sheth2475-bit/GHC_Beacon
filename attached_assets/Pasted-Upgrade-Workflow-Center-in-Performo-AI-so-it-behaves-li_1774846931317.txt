Upgrade Workflow Center in Performo AI so it behaves like a configurable workflow operating system.

The current Workflow Center already has:
- Home
- Workflow Builder
- Submissions
- My Requests
- Renewal Tracker
- Service Desk
- Workflow Analytics

also think automation set-up if user wants auto emails sent to remind

Keep the module name Workflow Center, but redesign the logic so each workflow type follows this lifecycle:

1. Workflow Builder = configure the workflow type
2. Records / operational view = create and manage records for that workflow type
3. Analytics view = analyze activity, status, overdue items, trends, and workload for that workflow type

Use only Performo AI branding.
Do not mention any third-party product names anywhere in the UI.

====================================================
1. CORE PRODUCT LOGIC
====================================================

Workflow Center should be a configurable workflow engine.

For each workflow type, the system should support:
- configuration
- record creation and management
- analytics

This should apply to:
- Service Tickets
- Recurring Tasks
- Licenses
- Certificates
- and any future workflow types added later

====================================================
2. WORKFLOW BUILDER MUST BECOME THE CONFIGURATION LAYER
====================================================

Workflow Builder should not only show templates.
It should be the configuration area where admin defines how each workflow type works.

For each workflow type, allow configuration of:

A. General setup
- workflow name
- description
- category
- icon
- department scope
- visibility rules

B. Ticket / record structure
- reference number prefix
- title field
- required fields
- optional fields
- custom fields
- attachments allowed or not

C. Status workflow
Admin can define statuses per workflow type.

Examples:

Service Ticket:
- New
- Assigned
- In Progress
- Pending
- Resolved
- Closed
- Escalated

Recurring Task:
- Scheduled
- Due Soon
- In Progress
- Completed
- Overdue

License / Certificate:
- Active
- Expiring Soon
- Renewal in Progress
- Renewed
- Expired
- Suspended if needed

D. Priority structure
- low
- medium
- high
- critical
or custom priority labels

E. Assignment rules
- assign to owner/team by default
- assign by department/category if applicable
- assign renewal owner
- assign escalation owner

F. Reminder and automation rules
- reminder before due date
- reminder before expiry date
- overdue reminders
- escalation rules
- recurring generation rules
- SLA alerts for tickets
- renewal alerts for licenses/certificates

G. SLA configuration for ticket-style workflows
Especially for Service Tickets:
- SLA start logic
- due response target
- due resolution target
- breach handling
- escalation timing

H. Form structure
- what fields appear when creating a record
- field order
- required fields
- custom field visibility

13. AUTOMATIONS
====================================================

Workflow Builder should allow automation rules such as:
- when ticket created -> assign owner
- when due date near -> remind owner
- when SLA breached -> escalate
- when recurring item completed -> generate next occurrence
- when expiry date near -> send renewal reminder
- when expired -> change status automatically
